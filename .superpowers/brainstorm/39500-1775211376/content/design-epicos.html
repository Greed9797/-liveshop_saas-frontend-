Design dos 3 Épicos — LiveShop CRM

Decisões consolidadas. Clique em cada seção para aprovar.

1

🗺️ CRM Geográfico

Stack

flutter_map + OpenStreetMap (já instalado, zero custo)

Filtros

Mapa full-screen. Botão "⚙️ Filtros" no canto abre BottomSheet com checkboxes multiselect por cor/status. Legenda fixa (card flutuante canto inferior direito).

Geocoding

CEP → lat/lng via Nominatim (OSM) — gratuito, sem API key. Chamada feita no backend no POST /clientes. Lat/lng salvo no banco uma única vez.

PINS DO MAPA
Negociação (Prospect)
Contrato Enviado
Ativo (gera faturamento)
Inadimplente
Recomendação
Clientes sem CEP → view lista (tab toggle no header)

Migrations necessárias

ALTER TABLE clientes ADD COLUMN cep VARCHAR(9), logradouro TEXT, cidade TEXT, estado CHAR(2), siga TEXT;
2

📋 Onboarding do Lead (Formulário)

Steps do formulário

Step 1 — Dados pessoais
Nome*, Celular*, Email, CPF
Step 2 — Dados comerciais
CNPJ, Razão Social, Fat. Anual, Nicho, Siga (TikTok/Insta)
Step 3 — Localização
CEP* → autocomplete cidade/estado → geocoding automático
Step 4 — Qualificação
"Já vende no TikTok Live?" (Sim/Não) + Revisão dos dados

Fluxo pós-salvar

SALVAR → POST /clientes (com geocoding)
↓
AUTO → Navega para ContratoScreen
RASCUNHO → Salva e volta ao mapa

Backend — proxy CEP

GET /v1/cep/:cep → ViaCEP + Nominatim

ViaCEP retorna endereço · Nominatim converte para lat/lng · Tudo salvo em uma chamada só

3

✍️ Motor de Contratos (Fase 1)

PDF dinâmico

Backend com pdf-lib (Node.js, ~2MB). Template base em .pdf com campos marcados. POST /contratos/:id/gerar-pdf preenche e retorna URL do arquivo.

Assinatura (Fase 1)

📱 Flutter signature package
• Área ampla (landscape no mobile)
• Checkbox obrigatório: "Li e aceito os termos"
• Captura: IP + timestamp no backend
• Tudo impresso no rodapé do PDF

Slot Fase 2 (ZapSign)

signature_type: 'pad' | 'zapsign'
external_signature_id: NULL (por ora)

Fluxo do contrato

1. Template preenchido com dados do cliente
2a. "Assinar Agora" → pad na tela → salva PNG da assinatura
2b. "Enviar por WhatsApp" → link do PDF → pin vira 🟡
3. Backend gera PDF final com assinatura + rodapé jurídico
4. Status → em_análise → AnaliseFinanceiraScreen

Migrations adicionais

contratos: +signature_type, +external_signature_id,
+signature_image_url, +signed_ip, +accepted_terms_at
