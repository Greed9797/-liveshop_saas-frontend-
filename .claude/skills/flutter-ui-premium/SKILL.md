# Flutter UI/UX Premium Design Skill

Create distinctive, production-grade Flutter interfaces that look like they were designed by a senior UI designer, not generated by AI. Every widget, screen, and interaction should feel intentional, polished, and premium.

## Design Thinking (Before Writing Any Code)

Before touching any widget, answer these questions:
- **Who uses this?** Business user on desktop? Field worker on mobile? Both?
- **What's the tone?** SaaS dashboard (clean, data-dense), consumer app (playful, engaging), enterprise (serious, trustworthy)?
- **What device?** Desktop-first? Mobile-first? Both? This determines your layout strategy.
- **What's the existing design system?** Check for existing `app_colors.dart`, `app_theme.dart`, `ThemeExtension`, design tokens before creating anything new.

## Flutter-Specific Design Rules

### Typography
- **NEVER** use default Roboto. Use `google_fonts` package with a distinctive font (Outfit, Plus Jakarta Sans, DM Sans, Manrope, Satoshi).
- Create a centralized `AppTypography` class with named styles: `h1`, `h2`, `h3`, `body`, `bodySmall`, `caption`, `label`, `kpiNumber`.
- Headings: negative `letterSpacing` (-0.02em to -0.01em) for tighter, more premium feel.
- KPI numbers: large (28-48px), SemiBold (w600-w700), with negative letter-spacing.
- Labels/captions: 11-12px, Medium weight, UPPERCASE with positive letterSpacing (0.02em).
- **CRITICAL**: Never hardcode colors in TextStyle. Use `context.colors.textPrimary` or theme-derived colors so dark/light mode works.

### Color System
- Use `ThemeExtension<AppColorsExtension>` pattern for dark/light mode support.
- Access via `context.colors.xxx` extension method on BuildContext.
- **Primary color** should have variations: `primary`, `primaryHover`, `primaryLightBg` (primary at 8-12% opacity).
- **Semantic colors**: success (green), error (red), warning (amber), info (blue) — each with light/dark variants.
- **Surface hierarchy**: `background` (page bg) → `cardBackground` (elevated surface) → `surfaceElevated` (hover/selected state).
- **Text hierarchy**: `textPrimary` (main) → `textSecondary` (supporting) → `textTertiary` (disabled/placeholder).
- **NEVER** use `Colors.white`, `Colors.black`, `Colors.grey` directly in widgets. Always use theme tokens.
- **NEVER** use purple, neon, or random colors unless explicitly part of the design system.

### Card Pattern (Universal)
Every card in the app MUST follow this pattern:
```dart
Container(
  padding: const EdgeInsets.all(20), // minimum 16px, prefer 20px
  decoration: BoxDecoration(
    color: context.colors.cardBackground,
    borderRadius: BorderRadius.circular(16), // standard for cards
    border: Border.all(color: context.colors.divider),
    // Light mode: subtle shadow
    // Dark mode: no shadow, rely on border
  ),
  child: Column(
    crossAxisAlignment: CrossAxisAlignment.start,
    mainAxisSize: MainAxisSize.min,
    children: [...],
  ),
)
```

For interactive cards, use `Material` + `InkWell` instead of `GestureDetector` + `Container`:
```dart
Material(
  color: context.colors.cardBackground,
  clipBehavior: Clip.antiAlias,
  shape: RoundedRectangleBorder(
    borderRadius: BorderRadius.circular(16),
    side: BorderSide(color: context.colors.divider),
  ),
  child: InkWell(
    onTap: onTap,
    hoverColor: context.colors.primary.withOpacity(0.04),
    child: Padding(
      padding: const EdgeInsets.all(20),
      child: content,
    ),
  ),
)
```
**NEVER** use `GestureDetector` + `Container(decoration)` for cards in Flutter Web — it causes rendering bugs in CanvasKit where child widgets may not paint correctly.

### Border Radius Standards
- Cards: `16px`
- Inputs/TextFields: `12px`
- Buttons: `10px`
- Chips/Filter pills: `20px` (full pill)
- Status badges: `6px`
- Chart bar tops: `4px`

### Buttons with Depth
Buttons should feel tactile, not flat:
```dart
ElevatedButton.styleFrom(
  backgroundColor: context.colors.primary,
  foregroundColor: Colors.white,
  elevation: 2,
  shadowColor: context.colors.primary.withOpacity(0.4), // colored shadow, NOT black
  padding: const EdgeInsets.symmetric(horizontal: 20, vertical: 14),
  shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(10)),
)
```
- Hover: elevation 4-6
- Pressed: elevation 1
- Destructive buttons: use `context.colors.error` with error-colored shadow
- Outline buttons: hover state with `primary.withOpacity(0.06)` background

### Status Badges
Translucent background + colored text, NEVER solid color badges:
```dart
Container(
  padding: const EdgeInsets.symmetric(horizontal: 10, vertical: 4),
  decoration: BoxDecoration(
    color: statusColor.withOpacity(0.12), // translucent!
    borderRadius: BorderRadius.circular(6),
  ),
  child: Text(
    label.toUpperCase(),
    style: TextStyle(
      fontSize: 11,
      fontWeight: FontWeight.w600,
      letterSpacing: 0.4,
      color: statusColor, // solid text color
    ),
  ),
)
```

### Monetary Values
Money should be visually prominent:
```dart
RichText(
  text: TextSpan(children: [
    TextSpan(
      text: 'R\$ ',
      style: TextStyle(
        fontSize: mainFontSize * 0.65, // symbol smaller
        fontWeight: FontWeight.w500,
        color: color.withOpacity(0.6),
      ),
    ),
    TextSpan(
      text: formattedNumber, // always use NumberFormat.currency(locale: 'pt_BR')
      style: TextStyle(
        fontSize: mainFontSize,
        fontWeight: FontWeight.w700,
        letterSpacing: -0.5,
        color: color,
      ),
    ),
  ]),
)
```
- **ALWAYS** use `NumberFormat.currency(locale: 'pt_BR', symbol: 'R\$')` for Brazilian currency.
- **NEVER** use `toStringAsFixed(2)` for displaying money.

## Responsiveness (CRITICAL)

Every screen MUST work on 3 breakpoints:

```dart
class Responsive {
  static bool isMobile(BuildContext context) => MediaQuery.of(context).size.width < 600;
  static bool isTablet(BuildContext context) => MediaQuery.of(context).size.width >= 600 && MediaQuery.of(context).size.width < 1024;
  static bool isDesktop(BuildContext context) => MediaQuery.of(context).size.width >= 1024;
}
```

### Sidebar/Navigation
- **Mobile (<600px)**: `Scaffold` with `Drawer` (hamburger menu). NO sidebar in body Row.
- **Tablet (600-1024px)**: Sidebar compacta (60-68px, only icons) OR collapsible.
- **Desktop (>1024px)**: Full sidebar (200-220px).

**CRITICAL**: In mobile mode, the sidebar MUST be a `Drawer`, NOT a `SizedBox` inside a `Row`. Having a fixed-width sidebar inside a Row on mobile causes `RIGHT OVERFLOWED BY XXX PIXELS` on every screen.

### Grids
- **Mobile**: 2 columns for KPI cards, metric cards, grid items.
- **Tablet**: 3-4 columns.
- **Desktop**: 4-5 columns.

Use `LayoutBuilder` to determine column count:
```dart
LayoutBuilder(
  builder: (context, constraints) {
    final columns = constraints.maxWidth < 600 ? 2 : constraints.maxWidth < 1024 ? 3 : 5;
    // ...
  },
)
```

### Text Overflow
Every `Text` widget in a responsive layout MUST have:
```dart
Text(
  content,
  maxLines: 1, // or 2 for descriptions
  overflow: TextOverflow.ellipsis,
  style: TextStyle(color: context.colors.textPrimary), // ALWAYS explicit color
)
```

## Dark / Light Mode

### Implementation
Use `ThemeExtension` pattern:
```dart
class AppColorsExtension extends ThemeExtension<AppColorsExtension> {
  final Color background;
  final Color cardBackground;
  final Color textPrimary;
  // ... all tokens

  factory AppColorsExtension.light() => AppColorsExtension(
    background: Color(0xFFFAF8F6), // warm off-white, NOT pure white
    cardBackground: Color(0xFFFFFFFF),
    textPrimary: Color(0xFF1A1A1A),
    // ...
  );

  factory AppColorsExtension.dark() => AppColorsExtension(
    background: Color(0xFF0A0A0A), // deep matte black
    cardBackground: Color(0xFF1A1A1A), // elevated surface
    textPrimary: Color(0xFFF5F5F5),
    // ...
  );
}

extension AppColorsX on BuildContext {
  AppColorsExtension get colors => Theme.of(this).extension<AppColorsExtension>()!;
}
```

### Rules
- **NEVER** leave a `Text` without explicit `color`. In dark mode, inherited text colors can be invisible.
- Light mode background: warm off-white (#FAF8F6), NOT pure white (#FFFFFF).
- Dark mode: cards (#1A1A1A) must be visually distinct from background (#0A0A0A).
- Shadows: visible in light mode, invisible/minimal in dark mode. Use border for elevation in dark.
- Semantic colors adjust per mode: success light=#22C55E, success dark=#4ADE80.

## Common Bugs to Avoid

### Overflow
- Every `Row` with `Text` children MUST have at least one `Flexible` or `Expanded` wrapping the text.
- Headers with title + filters/actions: use `Column` on mobile, `Row` on desktop.
- Buttons in tight spaces: use `Flexible`, reduce padding, reduce fontSize.

### Invisible Content
- If a card appears empty (white rectangle with no text), check:
  1. `TextStyle` has explicit `color:` property
  2. Card background color ≠ text color
  3. No `Opacity(0)`, `Visibility(false)`, or `ClipRect` hiding content
  4. `childAspectRatio` in GridView isn't too small (content clipped by grid cell)
  5. `SingleChildScrollView` inside a grid cell can push content out of view
- **Fix**: Use `Material` + `InkWell` instead of `Container(decoration)` for cards in Flutter Web.

### GridView childAspectRatio
- If card content is getting cut off, switch from `childAspectRatio` to `mainAxisExtent` (fixed height):
```dart
SliverGridDelegateWithFixedCrossAxisCount(
  crossAxisCount: columns,
  mainAxisExtent: 260, // fixed height instead of ratio
  crossAxisSpacing: 12,
  mainAxisSpacing: 12,
)
```

## Spacing Standards

- Page padding: `24px` (mobile `16px`)
- Between sections: `24px`
- Between cards in a section: `12-16px`
- Card internal padding: `20px` (mobile `14-16px`)
- Between label and value in a card: `8px`
- Between title and subtitle: `4px`

## Micro-Interactions

### Hover on Cards
```dart
InkWell(
  hoverColor: context.colors.primary.withOpacity(0.04),
  // Material automatically handles hover state
)
```

### Loading States
Use shimmer/skeleton loading, NOT spinners:
```dart
// Use shimmer package
Shimmer.fromColors(
  baseColor: context.colors.divider,
  highlightColor: context.colors.cardBackground,
  child: Container(
    height: 120,
    decoration: BoxDecoration(
      color: context.colors.divider,
      borderRadius: BorderRadius.circular(16),
    ),
  ),
)
```

### Animated KPI Numbers
```dart
TweenAnimationBuilder<double>(
  tween: Tween(begin: 0, end: realValue),
  duration: const Duration(milliseconds: 800),
  curve: Curves.easeOutCubic,
  builder: (context, value, _) => Text(
    NumberFormat.currency(locale: 'pt_BR', symbol: 'R\$').format(value),
    style: TextStyle(fontSize: 28, fontWeight: FontWeight.w700),
  ),
)
```

## Anti-Patterns (NEVER Do These)

- ❌ `Colors.white`, `Colors.black`, `Colors.grey` in widgets — use theme tokens
- ❌ `GestureDetector` + `Container(decoration)` for cards on Flutter Web — use Material + InkWell
- ❌ Purple, neon green, or random colors outside the design system
- ❌ Solid-color status badges (orange/red/green backgrounds with white text)
- ❌ `toStringAsFixed(2)` for money display — use NumberFormat
- ❌ Fixed-width sidebar in a Row on mobile — use Drawer
- ❌ Text without explicit color in TextStyle
- ❌ Cards without minimum 16px padding
- ❌ Inconsistent border-radius across the app
- ❌ Spinners for loading — use shimmer/skeleton
- ❌ Dashed/dotted borders on cards
- ❌ Giant colored banners taking up entire cards (chapado)
